Open Data Portal

Chicago's open data portal provides user-fiendly and machine-readable access to nearly 500 datasets. The public can browse data in a spreadsheet-like format, create maps, and graphics of Chicago's daily activities. The data portal is also a favorite for the city's wonderful civic-developer community, who uses this data to create helpful applications to connect the public to data.

Github

Chicago has begun releasing select datasets on the city's GitHub account. These data are provided in a business-friendly MIT license. By publishing data on GitHub, it also provides the community a chance to edit any changes or errors. As a result, Chicago is the only city on OpenStreetMap with a full list of all buildings.

Chicago Data Dictionary

Chicago's data dictionary will display all metadata, data about data, on each database in the city. While the data portal displays actual data, the data dictionary will provide a map for every database in the city, including information about data not published on the portal. The data dictionary is a key step to providing full transparency to the city's data.

FOIA Text Mining

The city regularly tracks Freedom of Information Act (FOIA) requests received by each department. The requests, with brief descriptions, are published on the data portal. The city's advanced analytics team will begin using text mining techniques to identify frequently-requested datasets through FOIA requests. Advancements on text mining algorithms can unlock the information within FOIA descriptions, helping to identify data that is frequently demanded by the public.

Improve feedback mechanisms on requesting new datasets

Portal users have been able to submit questions and requests for new datasets through the email address, dataportal@cityofchicago.org, and the Chief Data Officer's Twitter account, @ChicagoCDO. However, those mechanisms did not require uniform feedback and are not transparent for others. Chicago will launch a new platform to solicit suggestions for new datasets. The new platform will publicly display requests through the portal, highlighting common requests over time.

Improve metadata on existing and new datasets

Data on the portal often reflects intricate details of complex processes throughout the city. Simple items, such as street closures and payments, can contain elements and items that are not obvious to the general public. While data is public, it can fail to qualify as useful. Metadata, data about data, can help clarify what fields indicate, thereby providing usefulness to data. The city will improve metadata on the portal with the specific goal of increasing usability of existing datasets.
